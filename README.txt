多线程爬虫重试机制：（切记：不要使用第二种方法）
    重试机制的两种思想：
        1、推荐思想：异常重试思想中具有限制重试次数机制 (设置限定重试次数)
        2、特殊思想：永久重试机制 （使用队列，异常就丢回到队列实现永久重试） （不推荐使用）
    对于大多数的网站而言使用具有限制重试次数的多线程爬虫机制（因为：1、对于永久重试机制而言，如果后期出现一些其他异常，而这个异常时属于永久重试机制的范畴，这样是不允许的；2、对目标网站的压力很大，很不友好，因为后期出现特殊异常就会一直重试。占用资源）
    但是对于某些要求特别严格的爬虫项目（如：数据丢失率低、网站异常频繁、爬取量大），这样的爬虫项目我们就会想着永久重试（但是不推荐，该种情况的优化思想：其实是可以将重试次数设置大一点就ok的。）
    总结：使用第一种思想
